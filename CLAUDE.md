# CLAUDE.md

This repository is used to create and design Dungeons & Dragons campaigns and oneshots.

## Project Structure

```
campaigns/       — Full campaign arcs (each campaign gets its own subfolder)
oneshots/        — Standalone one-session adventures
resources/       — Shared reference material
  items/         — Magic items, loot tables
  lore/          — World lore and setting notes
  maps/          — Battle maps and world maps
  npcs/          — NPC profiles and stat blocks
  rules/         — D&D rules references and homebrew rules
```

## Conventions

- All written content uses **Markdown** (`.md` files).
- Each campaign or oneshot lives in its own subfolder under `campaigns/` or `oneshots/`.
- Shared material (recurring NPCs, world lore, rules) goes in `resources/`.
- Session notes, plot outlines, and encounter designs are kept as separate markdown files within each adventure folder.

## How Claude Should Help

- **Worldbuilding**: Flesh out lore, locations, factions, and history.
- **NPC Design**: Create NPC backstories, motivations, personality traits, and stat blocks.
- **Encounter Design**: Build balanced combat encounters, puzzles, and social challenges.
- **Session Planning**: Help outline session flow, key scenes, and branching plot points.
- **Rules Reference**: Look up or clarify D&D 5e rules; help design homebrew rules.
- **Items & Loot**: Design magic items, loot tables, and treasure hoards.
- **Maps**: Describe or outline map layouts for battles and exploration.

When creating content, match the tone and style of existing material in the repo. Keep stat blocks and mechanical details accurate to D&D 5e unless told otherwise.
